Overview
Hello, my name is Mike and I'm TieRack's developer.  I write and customize a small software package for the electronic component industry, and I originally built TieRack for my own use, to assist me while installing my software, training users, and providing support.  Geographically, my closest customer is several states away so all of the setup, service, training, and support work that I do has to be done remotely.

For each new installation of my product, I needed complete and reliable remote access to the application server in my customers' office, and usually a few of the desktops as well for training and other tasks.  The majority of my customers, however, lacked the technical know-how to open their corporate network for standard VNC or pcAnywhere access, or were understandably hesitant to do so, fearing that it might permit unauthorized access along with my own.

The remote connection aspect became the most troublesome part of the entire process, and in several cases I lost excellent sales opportunities simply because the prospective customer wasn't able to successfully configure their firewall for the access I needed to get them up and running.

The solution I came up with, the one that completely dissolved the problem and the one that I've been using ever since, is TieRack.

TieRack allows system administrators and support technicians immediate, secure, and reliable access to an array of configured client workstations.  Installed clients maintain persistent connections to TieRack, and allow rack managers full access and control regardless of location, network configuration, or intermittent connectivity.  The connections are initiated from within the client's network, so no configuration changes or security compromises need to be made.

Below is a list of the things that TieRack has to offer.  Give it a try, and let me know what you think.  I hope you find it remotely useful :)

Features
RemoteView TieRack includes a custom build of the open-source RealVNC engine, and uses its reverse connection feature to initiate remote VNC access regardless of the client machine's location or firewall configuration.

This single feature revolutionized the way I was able to provide installation, configuration, and troubleshooting services.  Instead of having to ask my customers' technical staffs to forward ports, they simply install a TiePin, enter the connection details I send, and the machines I need are immediately available for VNC access.
   
CommandView This utility allows managers to open a command line session to the client machine and execute commands, batch files, etc.

This is great for quick troubleshooting.  If you've ever had to explain to a non-technical person how to do a ping or ipconfig, you'll certainly appreciate this.
   
FileView TieRack includes full file system browsing and bi-directional file transfers, complete with copy/paste integration with Windows Explorer.

This one's obvious, and a must-have for remote support and management.  If you're constantly emailing files from one machine to another, this is definitely a better way.
          
 SQLView SQLView gives managers full SQL query and command access to data sources that would otherwise only be available from the client machine.  Query results can be viewed immediately, stored for transfer, and exported to one of several file formats ( currently .csv, .xls, and .txt).  Local files can be transferred and imported as tables, and any data set can be viewed and manipulated with several powerful tools for filtering, transforming, and organizing the data itself.

This has turned out to be a real time-saver.  When I realized how much time I was spending jockeying query results between systems, I saw the benefit in developing a method to streamline the process.

For instance, SQLView allows me to open a connection to a client's database, query for a result set, bring the data back to my local system as XML, import it to a holding database, filter and organize it, and export it to a report, all from a single screen.
